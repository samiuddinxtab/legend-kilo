# Nav Highlighting Bug Fix Plan

## Problem Analysis
The server-side implementation should work in both dev and production, but there's a logical bug in the `isActive` function.

### Root Cause
The `isActive` function on line 9 uses `startsWith`:
```javascript
return currentPath.startsWith(href);
```

This causes incorrect matching because:
- On `/blog/some-article`: `'/blog/some-article'.startsWith('/')` returns **true**
- This means **Home** link stays active for ALL pages since every path starts with `/`

### Expected Behavior
- Home (`/`) → exact match only
- Other pages (`/capabilities`, `/process`, etc.) → exact match OR sub-path match (e.g., `/blog/slug` should highlight Blog)

### Solution
Fix the `isActive` function to properly handle the root path:

```typescript
function isActive(href: string, currentPath: string): boolean {
  if (href === '/') return currentPath === '/';
  return currentPath === href || currentPath.startsWith(href + '/');
}
```

This ensures:
- Home `/` only matches exact `/`
- `/capabilities` matches `/capabilities` AND `/capabilities#section`
- `/blog` matches `/blog` AND `/blog/slug`
