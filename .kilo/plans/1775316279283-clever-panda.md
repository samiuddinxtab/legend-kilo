## Implementation Plan

### Phase 1: Legacy URL Remediation (HIGH PRIORITY)

#### Known Legacy URLs (Explicit Handling)
| URL | Action |
|-----|--------|
| `/yamaha-mt-15-launches/` | 410 Gone |
| `/new-yamaha-mt-15-launched/` | 410 Gone |
| `/yamaha-mt-15/` | 410 Gone |

#### Strategic Approach for Unknown Legacy URLs
Since full list is unknown, implement proactive detection:

| Strategy | Implementation | Priority |
|----------|---------------|----------|
| **Pattern-based catch-all** | Add `_redirects` rule for any URL containing "yamaha", "mt-15", "motorcycle", "bike" | High |
| **Log analysis** | Monitor 404s in Cloudflare analytics for unknown patterns | Medium |
| **sitemap.xml validation** | Ensure ONLY current site pages are in sitemap | High |

**Proposed wildcard rules for _redirects:**
```
/yamaha-mt-15-launches/  410
/new-yamaha-mt-15-launched/  410
/yamaha-mt-15/  410
/yamaha-*  410
/mt-15*  410
```

> ⚠️ **Pattern Matching Risk**: Wildcard rules like `/yamaha-*` and `/mt-15*` could inadvertently match future legitimate URLs. Recommend:
> 1. Test patterns in staging first
> 2. Start with specific known URLs only
> 3. Add wildcards later if unknown patterns surface in logs
